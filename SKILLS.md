# SKILLS.md

## 목적

이 문서는 수면 시차 PWA 작업에서 반복해서 사용할 실행 절차를 정리합니다. 패키지 매니저, 브랜치 전략, 코드 컨벤션, 폴더 구조 같은 프로젝트 전체 규칙은 `AGENTS.md`를 기준으로 따릅니다.

## 프로젝트 초기 세팅

새 프로젝트를 시작할 때는 아래 순서로 진행합니다.

1. Vite React TypeScript 프로젝트를 생성합니다.
2. npm 기반으로 의존성을 설치합니다.
3. ESLint, Prettier, Tailwind CSS를 설정합니다.
4. path alias를 설정합니다.
5. 기본 폴더 구조를 생성합니다.
6. Vite PWA Plugin을 설정합니다.
7. `.env.example`을 추가합니다.
8. GitHub 이슈 템플릿과 PR 템플릿을 추가합니다.
9. 빈 페이지라도 Vercel에 배포해 배포 흐름을 먼저 확인합니다.
10. `npm run lint`와 `npm run build`로 초기 상태를 검증합니다.

## MVP 1. 수면 시차 계산기 구현

수면 시차 계산기를 구현할 때는 아래 순서로 진행합니다.

1. 화면에 필요한 입력값을 정리합니다.
   - 현재 체류 국가 또는 도시
   - 현재 수면 시작 시간
   - 현재 기상 시간
   - 목표 수면 시작 시간
   - 목표 기상 시간
   - 기준 날짜
2. mock 위치 데이터를 먼저 만듭니다.
3. 수면 시간을 분 단위로 변환하는 유틸 함수를 작성합니다.
4. 자정을 넘는 수면 시간을 처리합니다.
5. 현재 수면과 목표 수면의 차이를 계산합니다.
6. 기준 도시와 체류지의 타임존 차이를 계산합니다.
7. 결과 카드 UI를 구현합니다.
8. 잘못된 입력값에 대한 에러 상태를 구현합니다.
9. 모바일 화면에서 입력 폼과 결과 카드가 자연스럽게 이어지는지 확인합니다.
10. `npm run lint`와 `npm run build`를 실행합니다.

## MVP 2. 친구 지도 구현

친구가 사는 나라 보기 기능을 구현할 때는 아래 순서로 진행합니다.

1. 친구 데이터 타입을 정의합니다.
2. mock 친구 데이터를 만듭니다.
3. 친구 추가 폼을 구현합니다.
4. 친구 위치 목록과 지도 또는 지도 대체 UI를 표시합니다.
5. 친구의 현재 시간을 계산합니다.
6. 친구의 수면 상태를 계산합니다.
   - 깨어 있음
   - 곧 잘 시간
   - 수면 중
   - 곧 일어날 시간
7. 친구 카드 UI를 구현합니다.
8. 친구가 없을 때의 빈 상태를 구현합니다.
9. 지도 API가 실패하거나 로딩 중일 때의 대체 UI를 구현합니다.
10. `npm run lint`와 `npm run build`를 실행합니다.

## MVP 3. 귀국 루틴 구현

귀국 루틴과 체크리스트를 구현할 때는 아래 순서로 진행합니다.

1. 귀국 루틴에 필요한 입력값을 정리합니다.
   - 현재 체류지
   - 귀국 날짜
   - 한국 도착 예정 시간
   - 현재 수면 시간
   - 목표 한국 수면 시간
   - 조정 강도
2. 귀국까지 남은 일수를 계산합니다.
3. 조정 강도별 하루 수면 이동 시간을 정의합니다.
4. 날짜별 수면 조정 루틴을 생성합니다.
5. 귀국 전 체크리스트의 기본 항목을 정의합니다.
6. 체크리스트 완료 상태를 저장합니다.
7. 진행률 UI를 구현합니다.
8. 귀국 당일 안내 문구를 구현합니다.
9. 로컬 저장소에 체크리스트 상태를 저장할지 검토합니다.
10. `npm run lint`와 `npm run build`를 실행합니다.

## 시간대 계산

시간대 관련 로직을 작성할 때는 아래 사항을 확인합니다.

1. 시간 계산은 문자열 상태로 직접 비교하지 않고 분 단위 또는 Date 객체로 변환합니다.
2. 수면 시작 시간이 기상 시간보다 늦으면 다음 날 기상으로 처리합니다.
3. UTC offset에 30분 또는 45분 단위가 있는 국가를 고려합니다.
4. 서울 기준 계산만 하드코딩하지 않고 기준 도시를 바꿀 수 있게 구성합니다.
5. 날짜 변경선을 넘어가는 케이스를 테스트합니다.
6. 브라우저의 현재 로컬 시간대와 사용자가 선택한 체류지 시간대를 구분합니다.

## 위치 검색과 지도 연동

위치 검색 또는 지도 기능을 구현할 때는 아래 순서로 진행합니다.

1. MVP 단계에서는 mock 도시 데이터로 먼저 구현합니다.
2. 실제 API를 붙이기 전에 필요한 필드를 정의합니다.
   - 국가명
   - 도시명
   - 타임존
   - 위도
   - 경도
3. API 키가 필요하면 `.env.local`과 `.env.example`에 변수를 추가합니다.
4. API 실패 상태와 빈 검색 결과 상태를 구현합니다.
5. 지도 위 마커가 겹치는 경우를 고려합니다.
6. 모바일 화면에서는 지도와 친구 카드의 높이를 명확히 제한합니다.

## PWA 설정

PWA 작업은 아래 순서로 진행합니다.

1. `vite-plugin-pwa`를 설치합니다.
2. `vite.config.ts`에 PWA 설정을 추가합니다.
3. 앱 이름, 짧은 이름, 설명, 테마 색상을 지정합니다.
4. 아이콘 파일을 `public`에 추가합니다.
5. manifest가 정상 생성되는지 확인합니다.
6. 기본 캐싱 전략을 설정합니다.
7. 오프라인 상태에서 보여줄 화면 또는 최근 계산 결과를 검토합니다.
8. 모바일 브라우저에서 설치 가능 여부를 확인합니다.

## 상태 관리

상태 관리는 아래 기준으로 선택합니다.

1. 한 화면 안에서만 쓰는 상태는 `useState`를 사용합니다.
2. 여러 화면에서 공유하는 클라이언트 상태는 Zustand를 사용합니다.
3. 친구 목록, 사용자 설정, 체크리스트 상태는 Zustand 후보입니다.
4. 실제 서버 API가 생기면 서버 상태는 별도 query 라이브러리 도입을 검토합니다.
5. 로컬 저장이 필요한 상태는 Zustand persist 또는 localStorage를 검토합니다.

## UI 검증

사용자에게 보이는 UI 작업을 마치기 전에 아래 항목을 확인합니다.

1. 모바일 화면에서 가로 스크롤이 생기지 않는지 확인합니다.
2. 데스크톱 화면에서 모바일 중심 레이아웃이 어색하게 늘어나지 않는지 확인합니다.
3. 터치 영역과 버튼 간격이 충분한지 확인합니다.
4. 입력 폼의 에러 메시지가 명확한지 확인합니다.
5. 결과 카드의 시간 정보가 한눈에 읽히는지 확인합니다.
6. 로딩, 빈 상태, 에러 상태가 필요한 화면인지 확인합니다.
7. `npm run lint`를 실행합니다.
8. `npm run build`를 실행합니다.

## 배포 준비

Vercel 배포 전 아래 순서로 확인합니다.

1. `.env.example`에 필요한 환경 변수가 모두 있는지 확인합니다.
2. 로컬에서 `npm run build`가 성공하는지 확인합니다.
3. Vercel 프로젝트를 연결합니다.
4. 필요한 환경 변수를 Vercel에 등록합니다.
5. 빈 페이지라도 먼저 배포해 URL을 확보합니다.
6. 배포 URL에서 PWA manifest, 아이콘, 기본 화면이 정상 호출되는지 확인합니다.
7. 이후 PR 단위로 preview deployment를 확인합니다.

## PR 준비

PR을 만들기 전에 아래 순서로 확인합니다.

1. 브랜치명이 프로젝트 규칙을 따르는지 확인합니다.
2. 작업 범위와 관련 없는 파일이 수정되지 않았는지 확인합니다.
3. `git status --short`로 변경 파일을 확인합니다.
4. `npm run lint`를 실행합니다.
5. `npm run build`를 실행합니다.
6. 변경된 화면, 컴포넌트, mock 데이터, 유틸 함수를 요약합니다.
7. 필요한 경우 스크린샷을 첨부합니다.
8. 관련 이슈 번호를 연결합니다.
